# x64 (64 位) 汇编指南

> 从 32 位 (x86) 升级到 64 位 (x86-64/x64) 的完整对比

---

## 第 1 部分：核心差异速览

| 特性 | 32 位 (x86) | 64 位 (x64) |
|------|-----------|-----------|
| **寄存器宽度** | 32 位 | 64 位 |
| **通用寄存器数** | 8 个 | 16 个 |
| **内存寻址** | 4GB | 16 EB（接近无限） |
| **参数传递** | 栈（cdecl） | 寄存器（System V ABI）|
| **返回值** | eax | rax（或 rdx:rax） |
| **编译指令** | `nasm -f elf32` | `nasm -f elf64` |
| **符号前缀** | 无 | `_` (某些系统) |
| **栈对齐** | 4 字节 | 16 字节 |

---

## 第 2 部分：寄存器名称变化

### 32 位 → 64 位 映射表

```
32 位  │ 64 位   │ 说明
───────┼─────────┼──────────────────────────
eax    │ rax     │ 返回值/累加器
ebx    │ rbx     │ 基址寄存器
ecx    │ rcx     │ 计数器/第1个参数
edx    │ rdx     │ 数据寄存器/第2个参数
esp    │ rsp     │ 栈指针
ebp    │ rbp     │ 帧指针
esi    │ rsi     │ 源指针/第3个参数
edi    │ rdi     │ 目标指针/第1个参数
───────┼─────────┼──────────────────────────
        │ r8     │ 第5个参数
        │ r9     │ 第6个参数
        │ r10    │ 临时寄存器
        │ r11    │ 临时寄存器
        │ r12    │ 临时寄存器
        │ r13    │ 临时寄存器
        │ r14    │ 临时寄存器
        │ r15    │ 临时寄存器
```

### 访问同一寄存器的不同大小

**以 rax 为例**：

```
┌─────────────────────────────────────────────────┐
│ rax (64 位)                                      │
├─────────────────────────────────────────────────┤
│ eax (32 位)       │ (高 32 位自动清零)          │
├───────┬───────────┤                             │
│ ax    │ 16 位     │ (低 16 位)                  │
├─┬─────┼──────┬────┤                             │
│ ah│ al│  ah │ al │ 8 位                        │
└─┴─────┴──────┴────┘                             │
```

**新增的 8 位访问**（只在 64 位中）：

```
r8  → r8d (32位) → r8w (16位) → r8b (8位)
r9  → r9d (32位) → r9w (16位) → r9b (8位)
...
r15 → r15d (32位) → r15w (16位) → r15b (8位)
```

## 第 3 部分：调用约定的巨大改变

### 32 位：cdecl（参数在栈上）

```
void add(int a, int b);

;; 调用
push 5               ; 第 2 个参数（最右边先 push）
push 3               ; 第 1 个参数
call add             ; 调用

;; 函数内部
add:
    push ebp
    mov ebp, esp
    mov eax, [ebp+8]   ; 第 1 个参数
    add eax, [ebp+12]  ; 第 2 个参数
    pop ebp
    ret
```

### 64 位：System V AMD64 ABI（参数在寄存器中）

```
void add(int a, int b);

;; 调用（Linux/Unix 风格）
mov rdi, 3           ; 第 1 个参数 → rdi
mov rsi, 5           ; 第 2 个参数 → rsi
call add             ; 调用

;; 函数内部
add:
    push rbp
    mov rbp, rsp
    mov eax, edi       ; 第 1 个参数（从 rdi）
    add eax, esi       ; 第 2 个参数（从 rsi）
    pop rbp
    ret
```

### 参数顺序对比

| 参数编号 | 32 位 (x86) | 64 位 Linux | 64 位 Windows |
|--------|-----------|-----------|---------------|
| 第 1 个 | 栈 [esp+4] | rdi | rcx |
| 第 2 个 | 栈 [esp+8] | rsi | rdx |
| 第 3 个 | 栈 [esp+12] | rdx | r8 |
| 第 4 个 | 栈 [esp+16] | rcx | r9 |
| 第 5 个 | 栈 [esp+20] | r8 | 栈 |
| 第 6 个 | 栈 [esp+24] | r9 | 栈 |

**本教程使用 Linux System V ABI**（大多数 Linux 系统）

## 第 4 部分：逐条指令迁移

### 指令语法基本相同

```asm
;; 32 位
mov eax, 100
add eax, 50
jmp label

;; 64 位（完全相同的语法！）
mov rax, 100
add rax, 50
jmp label
```

### 常用指令的 64 位版本

```asm
;; 数据移动
mov rax, 100         ; 64 位赋值
mov eax, 100         ; 32 位赋值（高 32 位清零！）
mov rax, qword [rbx] ; 从内存读 64 位

;; 算术
add rax, 50
sub rsi, rdi
imul rax, rcx        ; 64 位乘法
idiv rcx             ; 64 位除法
cqo                  ; 64 位符号扩展（替代 cdq）

;; 逻辑
shl rax, 4
shr rax, 2

;; 栈操作
push rax             ; 压入 64 位（8 字节）
pop rax
pushad               ; ❌ 不存在！（只在 32 位）
popad                ; ❌ 不存在！

;; 比较和跳转（完全相同）
cmp rax, rbx
je label
jl label
```

**⚠️ 重要**：`pushad/popad` 在 64 位中不存在！

## 第 5 部分：内存寻址的变化

### 32 位内存寻址

```asm
; 基本形式：[基址 + 索引*大小 + 偏移]
mov eax, [ebx]              ; 直接读
mov eax, [ebx + 4]          ; 加固定偏移
mov eax, [ebx + ecx*4]      ; 基址 + 索引*大小
mov eax, [ebx + ecx*4 + 8]  ; 完整形式
```

### 64 位内存寻址（基本相同）

```asm
; 基本形式：[基址 + 索引*大小 + 偏移]
mov rax, [rbx]              ; 直接读
mov rax, [rbx + 4]          ; 加固定偏移
mov rax, [rbx + rcx*4]      ; 基址 + 索引*大小
mov rax, [rbx + rcx*4 + 8]  ; 完整形式

; 但在 64 位中，可以使用新的寄存器
mov rax, [r8 + r9*8]        ; 使用 r8, r9 等
mov rax, [rsp + 16]         ; 相对于 rsp
```

### RIP 相对寻址（64 位新特性）

```asm
; 用于位置无关代码（PIC）
mov rax, [rel variable]     ; 相对于当前指令的位置
```

## 第 6 部分：编写 64 位汇编

### 最小的 64 位函数

```asm
; hello.asm（64 位）
bits 64

extern printf

section .data
    msg db "Hello x64!", 0

section .text
global main

main:
    push rbp
    mov rbp, rsp
    
    ; 在 Linux 上调用 printf("Hello x64!")
    mov rdi, msg        ; 第 1 个参数 → rdi
    call printf         ; printf 在第 1 个参数使用 rdi
    
    xor eax, eax        ; 返回 0（eax = 0，高 32 位自动清零）
    pop rbp
    ret
```

### 编译和运行

```bash
nasm -f elf64 hello.asm -o hello.o
gcc hello.o -o hello
./hello
```

## 第 7 部分：从 32 位迁移到 64 位

### 示例 1：简单赋值

**32 位**：
```asm
mov eax, 100
add eax, 50
```

**64 位**：
```asm
mov rax, 100         ; 改 eax → rax
add rax, 50          ; 改 eax → rax
```

### 示例 2：函数调用

**32 位（参数在栈上）**：
```asm
push 5
push 3
call my_add
add esp, 8
mov [result], eax
```

**64 位（参数在寄存器上）**：
```asm
mov rdi, 3           ; 第 1 个参数 → rdi
mov rsi, 5           ; 第 2 个参数 → rsi
call my_add          ; 调用
mov [result], rax    ; 返回值在 rax
```

### 示例 3：循环

**32 位**：
```asm
xor ecx, ecx
loop_start:
    cmp ecx, 10
    jge loop_end
    add [sum], eax
    inc ecx
    jmp loop_start
loop_end:
```

**64 位**：
```asm
xor rcx, rcx         ; 64 位计数器
loop_start:
    cmp rcx, 10
    jge loop_end
    add [sum], rax   ; 注意：sum 可能需要 mov rax, [sum] 然后 add
    inc rcx
    jmp loop_start
loop_end:
```

### 示例 4：完整函数（分支逻辑）

**32 位**：
```asm
your_if:
    mov eax, dword [a1]
    cmp eax, 12
    jge else_if_2
    
    mov eax, dword [a1]
    mov ecx, 2
    cdq
    idiv ecx
    add eax, 1
    mov dword [if_flag], eax
    jmp if_done
    
else_if_2:
    ; ...
if_done:
```

**64 位**：
```asm
your_if:
    mov eax, dword [rel a1]   ; rip 相对寻址（或用绝对地址）
    cmp eax, 12
    jge else_if_2
    
    mov eax, dword [rel a1]
    mov ecx, 2
    cdq                        ; 扩展 eax → rdx:rax
    idiv ecx                   ; ecx 是 32 位，会扩展到 rcx
    add eax, 1
    mov dword [rel if_flag], eax
    jmp if_done
    
else_if_2:
    ; ...
if_done:
```

## 第 8 部分：64 位特有的坑

### 坑 1：mov 的立即数限制

```asm
; 32 位：可以 mov 任意 32 位立即数
mov eax, 0x12345678

; 64 位：不能直接 mov 64 位立即数！
mov rax, 0x123456789ABCDEF0  ; ❌ 错误！

; 解决方案 1：分两步
mov rax, 0x12345678          ; 先高 32 位（自动清零低 32 位）
shl rax, 32
or rax, 0x9ABCDEF0           ; 再 or 低 32 位

; 解决方案 2：从内存读
mov rax, [big_number]

; 解决方案 3：使用 movabs（某些编译器支持）
movabs rax, 0x123456789ABCDEF0
```

### 坑 2：自动符号扩展

```asm
; 32 位赋值时，高 32 位被自动清零！
mov eax, 100        ; rax 变成 0x0000000000000064
mov eax, -1         ; rax 变成 0x00000000FFFFFFFF（⚠️ 注意）

; 64 位赋值时，根据值的大小决定
mov rax, 100        ; rax = 0x0000000000000064
mov rax, -1         ; rax = 0xFFFFFFFFFFFFFFFF（全 1）
```

### 坑 3：栈对齐要求

```
32 位：栈对齐 4 字节
64 位：栈对齐 16 字节（call 指令前）

; 64 位进入函数时，rsp 应该 % 16 == 0
; call 会 push 返回地址（8 字节），所以进入函数后 rsp % 16 == 8
```

### 坑 4：内存寻址需要注意大小

```asm
; 32 位：mov [addr], 100 自动是 32 位
mov [eax], 100

; 64 位：必须显式指定大小
mov qword [rax], 100         ; 64 位
mov dword [rax], 100         ; 32 位
mov word [rax], 100          ; 16 位
mov byte [rax], 100          ; 8 位
```

## 第 9 部分：实战对比

### 完整例子：求和数组

**32 位版本**：

```asm
; 参数在栈上
sum_array:
    push ebp
    mov ebp, esp
    
    mov eax, [ebp+8]   ; 第 1 参数：数组指针
    mov ecx, [ebp+12]  ; 第 2 参数：元素个数
    xor edx, edx       ; edx = 0（求和）
    xor esi, esi       ; esi = 0（索引）
    
loop:
    cmp esi, ecx
    jge loop_end
    add edx, [eax + esi*4]
    inc esi
    jmp loop
    
loop_end:
    mov eax, edx       ; 返回值
    pop ebp
    ret
```

**64 位版本**：

```asm
; 参数在寄存器上（rdi, rsi）
sum_array:
    push rbp
    mov rbp, rsp
    
    mov rax, rdi       ; 第 1 参数：数组指针（rdi）
    mov rcx, rsi       ; 第 2 参数：元素个数（rsi）
    xor rdx, rdx       ; rdx = 0（求和）
    xor r8, r8         ; r8 = 0（索引）
    
loop:
    cmp r8, rcx
    jge loop_end
    add rdx, [rax + r8*8]    ; 注意：64 位系统，数组大小乘以 8
    inc r8
    jmp loop
    
loop_end:
    mov rax, rdx       ; 返回值在 rax
    pop rbp
    ret
```

## 第 10 部分：建议

### 何时用 32 位？
- ✅ 学习阶段（如这个实验）
- ✅ 兼容旧系统
- ✅ 代码更简洁

### 何时用 64 位？
- ✅ 需要访问 > 4GB 内存
- ✅ 生产环境
- ✅ 现代 Linux/Unix 系统

### 对于你的实验
- 目前实验框架是 **32 位**（`nasm -f elf32`）
- 升级到 64 位需要：
  1. 改 Makefile（`-f elf64`，`-m64`）
  2. 改汇编代码（所有寄存器名改成 64 位）
  3. 改 C++ 代码（调用约定不同）

### 如果想自己试试 64 位

创建新文件 `student_64.asm`：

```asm
bits 64

extern a1
extern a2
extern my_random
extern your_string
extern if_flag
extern while_flag
extern print_a_char

section .text
global your_function

your_if:
    ; 改所有 eax/ebx/... → rax/rbx/...
    ; 改所有 cdq → cqo
    ; 改所有 dword [x] → qword [x]
    
your_while:
    ; 同上

your_function:
    ; 同上
```

然后编译：
```bash
nasm -f elf64 student_64.asm -o student_64.o
gcc -m64 -c test.cpp
gcc -m64 -o test student_64.o test.o
./test
```

## 总结表

| 操作 | 32 位 | 64 位 |
|------|------|------|
| 编译 | `nasm -f elf32` | `nasm -f elf64` |
| 链接 | `gcc -m32` | `gcc -m64` |
| 寄存器 | eax, ebx, ... | rax, rbx, ...（+ r8-r15） |
| 参数 1 | 栈 | rdi |
| 参数 2 | 栈 | rsi |
| 返回值 | eax | rax |
| 符号扩展 | cdq | cqo |
| 内存宽度 | 4GB | ~16EB |
| 函数序言 | `push ebp; mov ebp, esp` | `push rbp; mov rbp, rsp` |
