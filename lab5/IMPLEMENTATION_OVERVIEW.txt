================================================================================
TECHNICAL HIGHLIGHTS
================================================================================

Stack Frame Layout (x86):
  High Address
    [arg]          <- parameter to entry function
    [ret addr]     <- points to thread_exit()
    [entry addr]   <- entry function address
    [ebx/ecx/edx/esi/edi] <- general registers (0)
    [ebp]          <- base pointer (0)
    [eax]          <- accumulator (0) <- esp points here
  Low Address

Thread Pool Design:
  - Static allocation: no fragmentation, predictable memory
  - 10 threads × 4KB = 40KB for thread structures
  - 10 stacks × 4KB = 40KB for thread stacks
  - Total: 80KB fixed memory overhead

Thread Creation Flow:
  1. Allocate thread structure from pool
  2. Allocate stack from pool
  3. Initialize metadata (ID, name, state, priority)
  4. Set up stack frame (entry point, argument, registers)
  5. Return thread descriptor

Thread-Local Storage:
  - Global current_thread pointer
  - thread_current() returns current running thread
  - thread_set_current() updates current thread
  - Scheduler uses these for context switching
